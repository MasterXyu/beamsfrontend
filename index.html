Beams is a front-end Web framework designed primarily for multimedia-heavy websites. It is the flagship of a suite of open-source modules for developers who need to support video, photos, and other content across the widest range of screens and use cases possible.

The Beams suite is currently v0.1.0, an alpha release. The modules are still nascent, to be sure. But we believe we at least have a good start here, something we can show to other developers, begin to build a community around, and start iterating quickly.

A quick summary of the modules, including a link to each one's code repo on GitHub:

  • Beams. The front-end module, built in HTML5, CSS3, and jQuery v2.1.0. The visual design is based on a responsive 20-column, 4,000-pixel grid — approximately the physical screen size of a 48-inch TV. Yet the framework also scales down smoothly to any other size from desktop to tablet to smartphone as well.
  • Beams Server. A simple server written in Node.js/Express for running feature-rich sites created with the front-end module.
  • Beams Ads. An optional jQuery plugin for showing interstitial display ads. You can also control how often a user sees an ad and track views of the ads with Google Analytics, all while maintaining responsive display across different screen sizes for your sponsors too.

We're glad to get pull requests, bug reports, or any other feedback you have about Beams. Drop us an email anytime or visit us on GitHub.

Video belongs on Beams

Embedded video players can be a particularly finnicky design element of many Web pages. Not so with Beams, where content in a <video> or <iframe> tag is just as responsive as anything else.

Try it with this self-hosted HTML5 video...

Of course, Beams integrates with embedded players from the most popular third-party video services as well. To pull this off, we rely on the nifty jQuery plugin FitVids.js by Chris Coyier and his friends at Paravel, free to use under the innovative WTFPL.
